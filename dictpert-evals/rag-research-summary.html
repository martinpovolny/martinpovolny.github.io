RAG Quality Research — Techniques Summary

Comprehensive overview of RAG improvement techniques from recent research, organized by implementation complexity.

Prompt-Only Techniques (No New Infrastructure)

1. Self-RAG — Self-Critique Retrieval

Paper: Self-RAG: Learning to Retrieve, Generate, and Critique through Self-Reflection (Asai et al., ICLR 2024)

Problem: Model hallucinates citations or uses retrieved content without verifying relevance.

Solution: After each retrieval, LLM emits short critique sentence: "Does this retrieved entry support my intended citation?" Zero-shot approximation — no training needed.

Implementation: Add critique instruction to system prompt after tool result injection. Model self-evaluates retrieved content before using it.

Dictpert application: Prevent [[ČJA:word]] citations that never appeared in LOOKUP results.

2. IRCoT — Interleaving Retrieval with Chain of Thought

Paper: Interleaving Retrieval with Chain-of-Thought Reasoning for Knowledge-Intensive Multi-Step Questions (Trivedi et al., ACL 2023)

Problem: Model jumps into retrieval without planning, wastes lookups on irrelevant terms.

Solution: LLM emits [PLAN: ...] block BEFORE any search markers, declaring search strategy.

Implementation: Add planning step instruction to system prompt. Forces sequential reasoning for multi-dictionary questions.

Dictpert application: For "Jak se říká holce na Brněnsku?", model plans: "1) search ČJA for geographic variants, 2) search SNČJ for dialect headwords, 3) cross-reference"

3. ReAct — Reasoning + Acting

Paper: ReAct: Synergizing Reasoning and Acting in Language Models (Yao et al., ICLR 2023)

Problem: Unclear why model chose specific searches, hard to debug failures.

Solution: Before each search marker, LLM emits [THOUGHT: why I'm searching and what I expect].

Implementation: Add explicit thought markers to tool protocol. Stripped from display, logged for eval analysis.

Dictpert application: [THOUGHT: "holka" returned 0 in SNČJ; trying regional variant "holča"] → [LOOKUP: holča | sncj]

Retrieval Improvements

4. HyDE — Hypothetical Document Embeddings

Paper: Precise Zero-Shot Dense Retrieval without Relevance Labels (Gao et al., ACL 2023)

Problem: Vocabulary mismatch — user says "holka", entry says "koc" (Brno slang).

Solution: Generate a hypothetical dictionary entry that would answer the question, embed THAT instead of the raw query.

Implementation: Intercept VSEARCH query, call LLM to generate fake entry, embed the synthetic text.

Dictpert application: Query "Jak se říká holce na Brněnsku?" → generate "koc: brněnský slangový výraz pro děvče..." → embed → finds real "koc" entry.

5. Query2Doc — Query Expansion

Paper: Query2doc: Query Expansion with Large Language Models (Wang et al., EMNLP 2023)

Problem: Single search term misses entries indexed under synonyms.

Solution: Before issuing [FTSEARCH:], call LLM to expand to 3–5 synonyms/related headword forms, build FTS5 OR query.

Implementation: Pure prompt + dispatcher change in dict_db.py, no new infrastructure.

Dictpert application: "holka" → expand to ["holka", "holča", "děvče", "dívka", "ženská"] → ftsearch "holka OR holča OR děvče" | cja

6. BGE Cross-Encoder Reranking

Paper: C-Pack: Packaged Resources To Advance General Chinese Embedding (BAAI, 2023)

Model: BAAI/bge-reranker-v2-m3 (multilingual cross-encoder)

Problem: Vector search returns semantically similar but irrelevant entries in top-10.

Solution: After VSEARCH returns top-20, run cross-encoder over (query, headword+excerpt) pairs; return top-5 to LLM.

Implementation: Add as optional rerank=True flag in dicts/vector_db.py.

Dictpert application: Reduces false VSEARCH hits that waste LOOKUP rounds.

7. LaBSE — Language-Agnostic BERT Sentence Embedding

Paper: Language-agnostic BERT Sentence Embedding (Feng et al., ACL 2020)

Problem: Current paraphrase-multilingual-mpnet-base-v2 may be out-of-distribution for dialect and historical Czech.

Solution: Benchmark LaBSE (109 languages, trained on parallel corpora) vs current embedding model.

Implementation: Eval on 20 queries over 500 SPJMS entries. If LaBSE wins on top-10 recall, swap in dicts/vector_db.py.

Dictpert application: Better semantic matching for archaic and dialectal Czech terms.

8. RAPTOR — Recursive Abstractive Processing for Tree-Organized Retrieval

Paper: RAPTOR: Recursive Abstractive Processing for Tree-Organized Retrieval (Sarthi et al., ICLR 2024)

Problem: Individual entry embeddings don't capture cluster-level patterns (e.g., "all bird-sound interjections in ČJA").

Solution: Cluster entries by semantic similarity, generate one-sentence summaries per cluster, index alongside individual entries. VSEARCH hit on cluster summary guides LLM to issue LOOKUPs across the cluster.

Implementation: Pilot on SPJMS (most homogeneous — all geographic names). Cluster 25k entries → generate summaries → add to ChromaDB.

Dictpert application: "volání na ptáky" matches cluster summary → LLM gets hint to search volání na kachny, volání na slepice, etc.

Advanced Techniques

9. FLARE — Forward-Looking Active Retrieval

Paper: Active Retrieval Augmented Generation (Jiang et al., EMNLP 2023)

Problem: Model hallucinates before retrieval occurs.

Solution: Trigger retrieval mid-generation when model confidence drops below threshold.

Requirements: Logprob access from API (may not be exposed by all providers).

Dictpert application: Catch hallucination-in-progress, trigger emergency LOOKUP before synthesis completes.

10. SPLADE — Sparse Lexical and Expansion Model

Paper: SPLADE: Sparse Lexical and Expansion Model for First Stage Ranking (Formal et al., SIGIR 2021)

Paper: SPLADE v2: Sparse Lexical and Expansion Model for Information Retrieval (Formal et al., 2021)

Problem: BM25 FTS misses semantically related terms not in query.

Solution: Replace FTS5 BM25 with SPLADE — learned sparse retrieval via BERT vocabulary projection (implicit query expansion).

Model: naver/splade-cocondenser-selfdistil

Implementation: Heavy infrastructure change. Evaluate after HyDE/Query2Doc validate vocabulary-gap hypothesis.

Dictpert application: "holka" automatically expands to dialect variants without explicit synonym list.

11. Toolformer — Tool Use Fine-Tuning

Paper: Toolformer: Language Models Can Teach Themselves to Use Tools (Schick et al., NeurIPS 2023)

Problem: Model doesn't know when to LOOKUP vs FTSEARCH vs VSEARCH.

Solution: Collect (query → search-marker) pairs from successful eval conversations; fine-tune smaller model to learn tool-selection patterns.

Requirements: ~500+ labeled successful conversations.

Dictpert application: Long-term — train lightweight router model to predict optimal search strategy.

12. Adaptive-RAG — Query Routing

Paper: Adaptive-RAG: Learning to Adapt Retrieval-Augmented Large Language Models through Question Complexity (Jeong et al., NAACL 2024)

Problem: All queries get same heavy VSEARCH+FTSEARCH+LOOKUP pipeline even when simple LOOKUP suffices.

Solution: Train small classifier to predict query complexity → route to LOOKUP-only, FTSEARCH+LOOKUP, or full pipeline.

Dictpert application: Heuristic version first — emit only LOOKUP if user query contains short Czech-looking word; escalate on zero result.

Evaluation Frameworks

13. MT-Bench — LLM-as-Judge

Paper: Judging LLM-as-a-Judge with MT-Bench and Chatbot Arena (Zheng et al., NeurIPS 2023)

Application: Send final conversation + retrieved context to judge LLM with Czech dialectology rubric: "Are all cited headwords verified via LOOKUP?", "Does answer correctly reflect geographic distribution?"

Implementation: --judge flag in scripts/test_prompt.py. Store scores in messages table. ~€0.01/conv.

14. RAGAS — Retrieval-Augmented Generation Assessment

Paper: RAGAS: Automated Evaluation of Retrieval Augmented Generation (Es et al., EACL 2024)

Metrics: Answer Faithfulness, Context Precision, Context Recall

Implementation: Python package ragas. Retrieved docs = LOOKUP/FTSEARCH results; ground truth = manually verified headwords.

Dictpert application: Automated eval metrics over existing conversations.

GraphRAG

15. GraphRAG — Knowledge Graph Retrieval

Paper: From Local to Global: A Graph RAG Approach to Query-Focused Summarization (Edge et al., Microsoft Research, 2024)

Problem: Linear entry-by-entry search misses graph-structured relationships (village → okres → dialect zone).

Solution: Build geo_graph table (village → okres → dialect_zone); expose as [GEOLOOKUP:] tool.

Dictpert application: "Jak se říká X na Valašsku?" → geographic graph traversal finds all Valašsko villages → searches entries from those villages.

Czech NLP Specific

16. RobeCzech Cross-Encoder

Paper: RobeCzech: Czech RoBERTa, a Monolingual Contextualized Language Representation Model (Straka et al., TSD 2021)

Model: ufal/robeczech-base

Problem: Multilingual models handle Czech case/number/gender variation less accurately than Czech-specific BERT.

Solution: Fine-tune RobeCzech as cross-encoder reranker on (query, entry) pairs using labeled eval data.

Requirements: ~100 labeled pairs for first experiment.

Dictpert application: Better reranking for Czech morphological variants.

Priority Implementation Order

Based on effort/impact ratio:

  1. Self-RAG critique (prompt-only, 2h) — immediate citation quality boost
  2. IRCoT planning (prompt-only, 2h) — better multi-step reasoning
  3. Query2Doc FTS expansion (1 day) — vocabulary gap fix
  4. HyDE for VSEARCH (1 day) — semantic vocabulary gap fix
  5. BGE reranker (2 days) — VSEARCH precision
  6. LLM-as-Judge eval (1 day) — scalable A/B testing
  7. Czech morpho normalizer (2 days, already done: majka) — zero-result reduction
  8. LaBSE comparison (1 day) — embedding quality baseline
  9. RAGAS metrics (2 days) — automated eval
  10. RobeCzech reranker (1 week) — Czech-specific boost

Research-tier (longer horizon): - RAPTOR clusters - FLARE mid-generation retrieval - SPLADE learned sparse retrieval - Toolformer fine-tuning - Adaptive-RAG routing
