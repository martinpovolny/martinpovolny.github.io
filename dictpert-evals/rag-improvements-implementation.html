2. HyDE: Hypothetical Document Embeddings (Priority: 🔥 High)

Target problem: Vocabulary mismatch — users say "holka", entries say "koc"

Research: HyDE, ACL 2023

Implementation approach

Generate a hypothetical dictionary entry that would answer the query, embed THAT instead of the raw query.

Example:

Query: "Jak se říká holce na Brněnsku?"
↓
HyDE generates:
"koc: brněnský slangový výraz pro děvče, mladou ženu. 
Používá se v hanteci (brněnském slangu). 
Příklad: 'Viděl jsem tam jednoho koce.'"
↓
Embed the fake entry (not the query)
↓
Vector search finds real "koc ~ děvče [hantec.cz]"

def _generate_hypothetical_entry(query: str, dict_context: str = "") -> str:
    """
    Generate a hypothetical dictionary entry for HyDE embedding.

    Args:
        query: User's natural language question
        dict_context: Optional hint about dictionary type (dialect, standard, etc.)

    Returns:
        Synthetic dictionary entry (2-3 sentences)
    """
    from providers import complete

    prompt = f"""Představ si, že jsi lexikograf. Napiš krátké slovníkové heslo (2-3 věty), 
které by odpovídalo na tento dotaz:

"{query}"

{dict_context}

Heslo by mělo obsahovat:
- Headword (klíčové slovo)
- Stručný výklad (co to znamená)
- Kontext použití nebo příklad

Formát: "headword: výklad. Kontext/příklad."

Vrať POUZE slovníkové heslo, bez dalšího textu."""

    # Use fast model for HyDE generation
    hyde_entry = complete(prompt, model="claude-haiku-4-5", max_tokens=200)
    return hyde_entry.strip()

3. Query Rewriting / Multi-Query (Priority: 🔥 High)

Target problem: FTS recall failures — query uses words not in entry text

Research: Production RAG analysis (30-45% precision improvement)

Implementation approach

Rewrite user query into 3-5 alternative phrasings, search each in parallel, merge results with deduplication.

Example:

Query: "frazémy o pití alkoholu"
↓
Rewrites:
1. "frazémy pivo alkohol" (keyword extraction)
2. "idiomy o chlastání" (synonym substitution)
3. "rčení spojená s pitím" (formal paraphrase)
↓
FTSEARCH each variant → union results
↓
Deduplicate by headword → "mít oprátky", "přes žízeň", etc.

def _expand_query_for_fts(query: str, num_variants: int = 3) -> list[str]:
    """
    Generate query variants for multi-query FTS retrieval.

    Args:
        query: Original user query
        num_variants: Number of alternative phrasings (default: 3)

    Returns:
        List of query variants including the original
    """
    from providers import complete

    prompt = f"""Přeformuluj tento dotaz {num_variants} různými způsoby pro fulltextové vyhledávání:

Původní dotaz: "{query}"

Požadavky:
1. Použij synonyma (pivo → chlast, holka → děvče)
2. Zkus formální i hovorovou verzi
3. Extrahuj klíčová slova
4. Změň slovosled

Vrať pouze seznam {num_variants} variant, každou na samostatném řádku, BEZ číslování."""

    try:
        rewrites_text = complete(prompt, model="claude-haiku-4-5", max_tokens=150)
        variants = [v.strip() for v in rewrites_text.strip().split('\n') if v.strip()]
        # Ensure we got at least some variants
        if not variants:
            return [query]
        return [query] + variants[:num_variants]
    except Exception as e:
        print(f"Query expansion failed: {e}", file=sys.stderr)
        return [query]  # Fallback to original


def _merge_fts_results(results_lists: list[list[dict]], limit: int) -> list[dict]:
    """
    Merge FTS results from multiple query variants.

    Deduplicates by headword, keeps highest rank from any variant.

    Args:
        results_lists: List of result lists from different query variants
        limit: Max results to return

    Returns:
        Merged and deduplicated results
    """
    # Reciprocal rank fusion: score = sum(1 / (rank + 60))
    # Constants from RRF paper: k=60 is a good default
    seen = {}  # headword -> (best_score, result_dict)

    for results in results_lists:
        for rank, result in enumerate(results):
            headword = (
                result.get('headword') or 
                result.get('heslove_slovo') or 
                result.get('heslo') or 
                result.get('cached_head') or 
                ''
            )
            if not headword:
                continue

            score = 1.0 / (rank + 60)

            if headword not in seen or score > seen[headword][0]:
                seen[headword] = (score, result)

    # Sort by score descending
    merged = sorted(seen.values(), key=lambda x: x[0], reverse=True)
    return [result for _, result in merged[:limit]]

def fts_search(
    dictionary: str, 
    query: str, 
    limit: int = 20,
    use_query_expansion: bool = True  # NEW parameter
) -> list[dict]:
    """
    Full-text search with optional query expansion.

    Args:
        dictionary: Dict ID (e.g., 'cja', 'sncj')
        query: Search query
        limit: Max results
        use_query_expansion: If True, generate query variants and merge results

    Returns:
        List of entry dicts with 'snippet' field
    """
    conn = get_connection(dictionary)

    if not use_query_expansion:
        # Original single-query path
        return _fts_search_single(conn, query, limit)

    # NEW: Multi-query path
    query_variants = _expand_query_for_fts(query, num_variants=3)
    print(f"[FTS Multi-Query] Original: {query}")
    print(f"[FTS Multi-Query] Variants: {query_variants[1:]}")

    # Search each variant
    all_results = []
    per_variant_limit = max(10, limit // len(query_variants))

    for variant in query_variants:
        try:
            results = _fts_search_single(conn, variant, per_variant_limit)
            all_results.append(results)
        except Exception as e:
            print(f"FTS variant '{variant}' failed: {e}", file=sys.stderr)
            all_results.append([])

    # Merge with reciprocal rank fusion
    merged = _merge_fts_results(all_results, limit)
    print(f"[FTS Multi-Query] Merged: {len(merged)} results from {sum(len(r) for r in all_results)} total")

    return merged
